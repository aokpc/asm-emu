# C言語コンパイラの仕組み

## コンパイラの主要段階

### 1. 字句解析 (Lexical Analysis / Tokenization)
- ソースコードを意味のある単位（トークン）に分割
- 例: `int x = 10;` → `INT`, `IDENTIFIER(x)`, `ASSIGN`, `NUMBER(10)`, `SEMICOLON`

### 2. 構文解析 (Syntax Analysis / Parsing)
- トークンから抽象構文木（AST）を生成
- 文法規則に従ってコードの構造を理解

### 3. 意味解析 (Semantic Analysis)
- 型チェック、変数の宣言確認
- スコープ解決、関数呼び出しの検証

### 4. 中間コード生成
- AST から中間表現（IR）を生成
- プラットフォーム独立な形式

### 5. 最適化
- デッドコード除去、定数畳み込み、ループ最適化など

### 6. コード生成
- 中間表現からターゲットマシンのアセンブリコードを生成

## 簡易版C→アセンブリ変換の方針

今回の実装では以下の段階に簡略化：
1. 字句解析（トークン化）
2. 構文解析（単純なパーサー）
3. コード生成（アセンブリ出力）

### 対応する基本構文
- 変数宣言: `int x;`
- 代入: `x = 10;`
- 算術演算: `+`, `-`, `*`
- 関数: `printf()` など
- 制御構文: `if`, `while`, `for`
- 関数定義と呼び出し

### メモリレイアウト
- 変数はメモリアドレス 0x10 から順に配置
- スタックは上位アドレス（0xFF）から下位に向かって使用
