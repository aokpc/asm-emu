# 解説
## レジスタとフラグ
このCPUには、いくつかの重要なレジスタとフラグがあります。

### pc (Program Counter) - プログラムカウンタ
役割: 次に実行する命令が格納されているコード上の行番号（アドレス）を指します。
動作: 通常、命令を1つ実行するごとに自動的に1増加します。jmpやcallなどの分岐命令が実行されると、指定されたアドレスに直接書き換えられます。
### zf (Zero Flag) - ゼロフラグ
役割: 演算結果がゼロであったかどうかを示すフラグです。
状態:
1: 直前の演算結果が 0 であったことを示します。
0: 直前の演算結果が 0 以外であったことを示します。
影響を受ける命令: add sub inc dec and or xor not neg lsl lsr sbc adc mul pop
用途: jz (Jump if Zero) や jnz (Jump if Not Zero) などの条件分岐命令で、処理の流れを変えるために使用されます。
### cf (Carry Flag) - キャリーフラグ
役割: 演算の結果、桁あふれ（キャリー）や桁借り（ボロー）が発生したことを示すフラグです。
状態:
1: キャリーまたはボローが発生したことを示します。
0: 発生しなかったことを示します。
詳細な動作:
加算 (add adc): 8ビットの演算で結果が255を超える（桁あふれする）場合に 1 になります。
減算 (sub sbc cmp): A - B の計算で A < B となり、上位ビットから借りてくる（ボロー）必要がある場合に 1 になります。
シフト (lsl lsr): シフト操作によって押し出されたビットの値を保持します。
用途: jc (Jump if Carry) や jnc (Jump if Not Carry) での条件分岐や、adc (Add with Carry) や sbc (Subtract with Carry) を使った多バイト演算に不可欠です。
## 命令セット
コードは以下のように記述します。カンマ、コメント、インデントは禁止です
```
main:
mov 0x10 "A"
```
### オペランド（引数）の記法：

dest: 結果を格納するメモリアドレス。メモリアドレス値またはメモリアドレスポインタ
src: 操作に使用する値。即値またはメモリアドレス値またはメモリアドレスポインタ
label: ジャンプ先のラベル名。
val: 値。即値またはメモリアドレス値またはメモリアドレスポインタ
addr: メモリアドレス。
### 値の指定方法：

0x10: メモリアドレス0x10に格納されている値。
#10: 即値 10。
"A": 文字 'A' の文字コード (65)。
*0x10: メモリアドレス0x10に格納されている値をポインタ（アドレス）として、そのポインタが指す先のメモリに入っている値（間接参照）。
### データ転送命令
### mov dest src

説明: srcの値をdestが示すメモリアドレスにコピーします。
例: mov 0x10 #5 (メモリアドレス0x10に値5を書き込む)
### push val

説明: valの値をスタックに積みます。スタックポインタ(sp)がデクリメントされます。
例: push #10
### pop dest

説明: スタックから値を取り出し、destが示すメモリアドレスに格納します。スタックポインタ(sp)がインクリメントされます。
例: pop 0x20
算術演算命令
### add dest src

説明: destの値とsrcの値を加算し、結果をdestに格納します。
フラグ: zf (結果が0の場合) cf (桁あふれした場合)
### sub dest src

説明: destの値からsrcの値を減算し、結果をdestに格納します。
フラグ: zf (結果が0の場合) cf (ボローが発生した場合)
### adc dest src

説明: destの値、srcの値、そしてcfの値を加算し、結果をdestに格納します。多バイト加算に使用します。
フラグ: zf cf
### sbc dest src

説明: destの値からsrcの値とcfの値を減算し、結果をdestに格納します。多バイト減算に使用します。
フラグ: zf cf
### inc dest

説明: destの値を1増やします。
フラグ: zf
### dec dest

説明: destの値を1減らします。
フラグ: zf
### mul dest src

説明: destの値とsrcの値を乗算し、結果の下位8ビットをdestに格納します。
フラグ: zf
### neg dest

説明: destの値の符号を反転します（2の補数を求めます）。
フラグ: zf
論理・ビット演算命令
### and dest src

説明: destとsrcでビットごとの論理積をとり、結果をdestに格納します。
フラグ: zf
### or dest src

説明: destとsrcでビットごとの論理和をとり、結果をdestに格納します。
フラグ: zf
### xor dest src

説明: destとsrcでビットごとの排他的論理和をとり、結果をdestに格納します。
フラグ: zf
### not dest

説明: destの全ビットを反転します。
フラグ: zf
### lsl dest (Left Shift Logical)

説明: destの値を1ビット左にシフトします（2倍）。最上位ビットはcfに格納されます。
フラグ: zf cf
### lsr dest (Right Shift Logical)

説明: destの値を1ビット右にシフトします（1/2倍）。最下位ビットはcfに格納されます。
フラグ: zf cf
### 比較・分岐命令
### cmp dest src

説明: destの値とsrcの値を比較（内部的に減算）し、結果をフラグに反映させます。値は変化しません。
フラグ: zf (dest == srcの場合) cf (dest < srcの場合)
### jmp label

説明: labelで示されるアドレスに無条件でジャンプします。
### jz label (Jump if Zero)

説明: zfが1の場合にlabelにジャンプします。
### jnz label (Jump if Not Zero)

説明: zfが0の場合にlabelにジャンプします。
### jc label (Jump if Carry)

説明: cfが1の場合にlabelにジャンプします。
### jnc label (Jump if Not Carry)

説明: cfが0の場合にlabelにジャンプします。
サブルーチン命令
### call label

説明: 戻り先アドレス（現在のpc + 1）をスタックに積み、labelにジャンプします。
### ret

説明: スタックから戻り先アドレスを取り出し、そのアドレスにジャンプしてサブルーチンから復帰します。
その他
### #string "text" addr

説明: 指定した文字列"text"を、addrから始まるメモリ領域に書き込みます。文字列の末尾には自動的にヌル文字(0)が追加されます。これはアセンブラに対する指示であり、実行サイクルは消費しません。
### printn val

説明: valの値を数値として出力エリアに表示します。
### printc val

説明: valの値を文字コードと解釈し、対応する文字を出力エリアに表示します。
